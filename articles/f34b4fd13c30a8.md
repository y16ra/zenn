## はじめに

バックエンドエンジニアとして Go や AWS を日常的に使っています。
業務・趣味で AI エージェント系のツールに触れる機会が増えてきました。それをきっかけに、体系的に知識を整理したいと思い、AWS Certified AI Practitioner（AIF-C01）の受験を決めました。

この記事では、試験の概要と勉強中に感じたことをまとめます。

---

## AIF-C01 とはどんな試験か

特定の職務に関係なく、AI/ML、生成 AI テクノロジー、関連する AWS のサービスとツールに関する総合的な知識を効果的に実証できる個人を対象としています。

- 一般的な、また AWS 上での、AI、ML、生成 AI の概念、手法、戦略を理解する。
- AI/ML と生成 AI テクノロジーの適切な使用を理解し、受験者の組織内で関連する質問を行う。
- 特定のユースケースへの適用に適した AI/ML テクノロジーの種類を見極める。
- AI、ML、生成 AI テクノロジーを責任を持って使用する。


### 基本情報

| 項目 | 内容 |
|------|------|
| レベル | ファンダメンタル（入門） |
| 対象者 | AWS の AI/ML に最大6か月程度携わった方（構築は必須ではない） |
| 合格点 | 700点（スケールスコア 100〜1,000） |
| 問題数 | 採点対象 50問 ＋ 非採点 15問 |
| 問題形式 | 単一選択、複数選択、並び替え、内容一致、ケーススタディ |

### ドメイン別出題比率

| ドメイン | 比率 | 主な内容 |
|---------|------|----------|
| 1. AI/ML の基礎 | 20% | 用語、ユースケース、ML 開発ライフサイクル・MLOps |
| 2. GenAI の基礎 | 24% | 生成 AI の概念、可能性と限界、AWS インフラ・サービス |
| 3. 基盤モデルの応用 | 28% | モデル選定、RAG、プロンプト、ファインチューニング、評価 |
| 4. 責任ある AI | 14% | バイアス・公平性、説明可能性、ガードレール等 |
| 5. セキュリティ・コンプライアンス・ガバナンス | 14% | IAM、暗号化、ガバナンス、規制対応 |

ドメイン3「基盤モデルの応用」は比重が最も高く、Amazon Bedrock を中心とした実践的な知識が問われます。

---

## バックエンドエンジニア目線で見た試験の特徴

- コーディングやアーキテクチャ設計は問われない（試験ガイド上も「モデル開発・データエンジニアリング・ハイパーパラメータチューニング・パイプライン構築」などは対象外）
- AI/ML の概念（教師あり学習、RAG、プロンプトエンジニアリングなど）を「使う側」として理解する試験
- AWS サービスの引き出しが多いほど選択肢を絞りやすい
- 普段 Bedrock や SageMaker を触っていると、問題の意図が掴みやすく有利になりそうだと感じている

## 試験範囲の主要 AWS サービス早見表

試験ガイド付録「範囲内の AWS のサービスと機能」に基づく主なサービスです。網羅ではなく、学習の目安として抜粋しています。

| カテゴリ | 主なサービス |
|---------|------------|
| GenAI・基盤モデル | Amazon Bedrock、Amazon Q、Amazon SageMaker（JumpStart 含む） |
| AI/ML マネージド API | Comprehend、Transcribe、Translate、Lex、Polly、Rekognition、Textract、Kendra、Personalize、Fraud Detector |
| データベース（RAG/ベクター等） | OpenSearch Service、Amazon Aurora、Amazon RDS、Amazon Neptune、Amazon DocumentDB |
| データ・分析 | AWS Glue、Glue DataBrew、Lake Formation、EMR、Redshift、QuickSight、Data Exchange |
| セキュリティ | IAM、KMS、Macie、PrivateLink、Secrets Manager |
| コンプライアンス・ガバナンス | CloudTrail、AWS Config、Audit Manager、AWS Artifact、Trusted Advisor、Well-Architected Tool |
| 人間の関与・説明可能性 | Amazon A2I、SageMaker Clarify、Model Monitor、Bedrock ガードレール、Model Cards |
| その他 | EC2、ECS、EKS、S3、CloudWatch、Inspector |

詳細な各サービスの解説は別途まとめ記事を書く予定です。

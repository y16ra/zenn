## Frontier Agentsとは何か

![Frontier Agents](/images/articles/c6b54eba81ceb8/intro_frontier_agents.png)

AWSが定義する**Frontier Agents**とは、**現在のものよりもはるかに高い能力を表す、能力における段階的な変化を表す新しいクラスのエージェント**です。次世代のAIエージェントとして、以下の3つの特徴を持ちます：

- Autonomous（自律的）
- Scalable（大規模な拡張可能）
- Long-running（長時間実行可能）

https://aws.amazon.com/jp/ai/frontier-agents/

## 発表された3つのFrontier Agents

![Frontier Agents](/images/articles/c6b54eba81ceb8/frontier_agents.jpg)

今回発表された3つのFrontier Agentsは以下の通りです。
- Kiro Autonomous Agent
- AWS Security Agent
- AWS DevOps Agent

それぞれのエージェントについて詳しく紹介していきます。

### Kiro Autonomous Agent

**Kiro Autonomous Agent**は、自律的なソフトウェア開発・メンテナンスにフォーカスしたエージェントです。

#### 主な機能
- **自律的な動作**: 開発タスクを自律的に処理し、あなたが他の作業に集中している間や離れている間も作業を進めます。
- **コンテキストの維持**: タスク、リポジトリ、PRにわたってコンテキストを維持し、途中から再開する際もコンテキストを再構築する必要がありません。
- **マルチリポジトリ対応**: 変更を一度計画すれば、複数のリポジトリにわたって調整された編集とPRを作成します。
- **並列実行**: 隔離されたサンドボックス環境でタスクを実行し、レビュー用のPRを作成するため、自分の作業をブロックせずに進捗できます。
- **学習能力**: コードベース、チケット、レビューフィードバックから継続的に学習し、チームの開発パターンに合わせた提案を行います。
- **安全なPR運用**: PRを作成してレビューを待ちますが、自動的にマージすることはありません。

#### チーム向け機能
チーム版では、Jira、Confluence、GitLab、GitHub、Teams、Slackなどのツールと連携し、チーム全体での並行開発をサポートします。

https://kiro.dev/autonomous-agent/

#### 開発効率の劇的な向上の実績

![Re-architectureプロジェクトの実績](/images/articles/c6b54eba81ceb8/re-architecture.png)

Keynoteでは、30人規模で18ヶ月かかる想定だった大規模な re-architecture プロジェクトを、**わずか6人で76日間**で完成させた事例が紹介されました。これは初期のGitHub Copilotなど、主にインライン補完を提供するAIコーディングツールで見られた10〜20%の効率向上とは、**桁違いの効率向上**です。

#### 所感
複数のリポジトリにまたがる大規模なライブラリのアップデートやプルリクエストの作成を並行して行うことができるというのは、非常に便利そうです。
また実績紹介で示された内容についてこれだけの効率向上ができるというのは驚きでした。

### AWS Security Agent

**AWS Security Agent**は、セキュリティに特化したFrontier Agentです。最初からセキュアなアプリケーションを構築するのを支援し、セキュリティの専門知識を上流に組み込みます。約20年にわたるAWSクラウドとアプリケーションセキュリティの専門知識とAWSセキュリティのベストプラクティスを活用して、包括的なセキュリティガイダンスを提供します。

#### 主な機能
- **設計段階からの介入**: コードを1行も書く前に、設計ドキュメントから問題を発見できます。
- **設計セキュリティレビュー**: 設計ドキュメントにリアルタイムのセキュリティフィードバックを提供し、コードが記述される前に組織のセキュリティ要件への準拠状況を評価します。
- **コードセキュリティレビュー**: プルリクエストを組織のセキュリティ要件や一般的な脆弱性と照らし合わせて分析し、修正ガイダンスを提供します。
- 継続的なセキュリティスキャンを実施します。
- **オンデマンドのペネトレーションテスト**: 専門のAIエージェントをデプロイして検証済みのセキュリティ脆弱性を発見・報告します。OWASPトップ10の脆弱性とビジネスロジックの欠陥に対してテストします。
- 脆弱性の検出と修正提案を行います。検出結果には、影響分析、再現可能な攻撃経路、すぐに実装できるコード修正が含まれます。
- **並列実行**: 複数のSecurity Agentを並行して起動し、すべてのアプリケーションをテストできます。
- **マルチクラウドとハイブリッドクラウドのサポート**: AWS環境、ハイブリッド環境、マルチクラウド環境にわたって動作し、一貫したセキュリティガイダンスとテストを提供します。
- **組織の要件を自動的に検証**: 組織のセキュリティ要件をAWSコンソールで一度定義すると、セキュリティレビューの度に特定のポリシーを使用してすべてのアプリケーションにわたって自動的に検証します。
- **アプリケーションに合わせたセキュリティガイダンス**: 設計ドキュメント、ビジネス要件、ソースコードを分析して各アプリケーションを把握し、テクノロジーパターン、アーキテクチャに関する決定、ビジネスコンテキストを理解することで、各アプリケーション固有のリスクプロファイルに合わせたガイダンスを提供します。
- **インテリジェントなアプリケーションコンテキスト**: ソースコードとドキュメントから学習してアプリケーションのコンテキストを把握し、自動セキュリティスキャンツールでは検出できない脆弱性を特定して利用します。アプリケーションのコンテキストとデータフローを把握することで、実際に悪用可能なリスクを検証する標的型攻撃シナリオを作成します。

https://aws.amazon.com/jp/security-agent/

#### 所感
これらの機能を活用できると、常に自動的にセキュリティチェックを行うことで、セキュリティの品質を高めることができそうです。また、セキュリティチームなどで手動で行っていた定期的なレビューやスキャンが自動化されることで、エンジニアはより創造的な業務にフォーカスできるようになりそうです。

### AWS DevOps Agent

**AWS DevOps Agent**は、インシデント対応・信頼性向上を目的としたFrontier Agentです。インシデントを解決し、積極的に予防することで、信頼性とパフォーマンスを継続的に向上させます。

#### 主な機能
- **自動調査とインシデント調整**: ServiceNowなどのチケットシステムやアラームと統合し、インシデントチケットから自動的に調査を開始します。インタラクティブなチャットやServiceNow、Slackなどのコラボレーションツール内で直接作業し、発見事項を共有して対応を調整します。必要に応じて、調査から直接AWSサポートケースを作成することもできます。
- **根本原因分析（RCA）**: オブザーバビリティツール、コードリポジトリ、CI/CDパイプラインと連携し、テレメトリ、コード、デプロイデータを相関・分析して、システム変更、入力の異常、リソース制限、コンポーネントの故障、依存関係の問題など、環境全体での根本原因を特定します。
- **詳細な緩和計画**: 根本原因を特定すると、事故を解決するための対応、成功の確認、必要に応じた変更の元に戻す手順を含む詳細な緩和プランを提供します。
- **アプリケーションリソースマッピング**: 環境について学習するにつれて、コンテナ、ネットワークコンポーネント、ロググループ、アラーム、CI/CDデプロイメントなどのアプリケーションリソースを特定し、それらがどのように接続されているかをマッピングして、アプリケーションリソースマップを作成します。これにより、リソースのトポロジーとテレメトリ、コード、デプロイデータを組み合わせて、問題の根本原因を正確に特定することができます。
- **Observabilityソリューションとの統合**: Amazon CloudWatch、Dynatrace、Datadog、New Relic、Splunkなどの既存のObservabilityソリューション、GitHub Actions、GitLabワークフローなどのコードリポジトリ・CI/CDパイプラインとの組み込みインテグレーションを提供します。
- **予防的な推奨事項**: 過去のインシデントのパターンを分析し、可観測性、インフラ最適化、デプロイメントパイプラインの改善、アプリケーションの回復力という4つの主要分野を強化するための実用的な推奨事項を提供します。また、将来のインシデントを防ぐためのCI/CDガードレールを推奨します。
- **継続的な学習**: 学習ループを使用して、推奨事項の改善を続け、運用上の優先事項に合わせ、チームのフィードバックに基づいて、組織のニーズにより適した関連性の高い推奨事項を提供します。

#### インシデント対応の実現内容
AWS DevOps Agentは、インシデント発生時にオンコール・エンジニアが確認する前に即座に対応を開始します。アプリケーション・トポロジーの知識と、異なるコンポーネント間の関係性を使用して、アラートから独立して遡って問題の根本原因を見つけます。Observabilityソリューションとの統合により、インシデントに関するより多くのコンテキストを提供し、問題を発見して修正案を提示します。オンコール・エンジニアがログインする頃には、DevOps Agentはすでに問題を発見し、変更を提案し、レビューと承認の準備ができています。また、将来のインシデントを防ぐため、CI/CDガードレールを推奨します。

https://aws.amazon.com/jp/devops-agent/

#### 所感
深夜の障害アラートに対して、エージェントが初期調査を完了させ、「ここが原因の可能性が高い。この修正案はどうか」と提示してくれるのはエンジニアにとって理想的なことだと思います。
DevOps Agentを活用してエージェントが自動的に対応を開始し、オンコールでエンジニアがログインする頃には、すでに問題を発見し、変更を提案し、レビューと承認の準備ができているということが実現したら最高だなと思いました。

## Frontier Agentsを支える基盤

Frontier Agentsは、AWSの強力なAIインフラの上に構築されています。
特に、**Bedrock AgentCore**の上に構築されており、エンタープライズレベルの安全性と信頼性を実現しています。

### Amazon Bedrock AgentCore

Frontier Agentsは**Bedrock AgentCore**の上に構築されており、エージェントを**安全に・本番品質で動かす**ための機能群が提供されています。

| 機能 | 説明 |
|------|------|
| **Agent Policy** | エージェントの動作範囲を制限するガードレールです。エージェントがツールやデータにアクセスするための適切な権限を定義し、リアルタイムで確定的なコントロールを提供します。 |
| **Agent Evaluations** | エージェントの出力品質を継続的に評価します。実世界の動作に基づいてエージェントの品質を継続的に検査します。 |
| **Memory & State** | 長時間実行のためのコンテキスト管理です。短期および長期の両方のメモリを処理しながらコンテキストを保持します。 |

これらの機能により、「エージェントが暴走しないか」「品質を担保できるか」というこれまでの懸念点を解消し、エージェントを本番環境で安全に運用できるようになりそうです。

https://aws.amazon.com/jp/bedrock/agentcore/

## おわりに

今年初めて参加したAWS re:Inventは、多くの刺激を与えてくれたイベントでした。AIが単なる「支援ツール」から、自律的に複雑なタスクを遂行するエージェントへと進化し、エージェントに対するガードレール機能や品質を評価できる機能が整ってきて実用的な段階に入ってきたことを強く感じました。

特に、Frontier Agents の発表は、私たちの働き方を根本から変える可能性を秘めていると感じました。Kiro Autonomous Agentによってコードを書く時間は短縮され、Security AgentやDevOps Agentを活用することでルーチンワークを自動化し、オンコール対応の負担も大幅に軽減されることが期待できます。開発者の仕事は、より戦略的で創造的な業務へとシフトしていくと考えられます。

また、セッションで何度も目にした Kiro を改めて触ってみたところ、spec-driven（仕様駆動）でタスクを進める方式の体験が秀逸でした。他のAIエージェント搭載エディタが最近搭載し始めたPlanモード機能よりも使いやすく感じたので、実際の業務でも試してみたくなりました。

AIエージェントの進化は加速しており、実際に手を動かして試すことが最良のキャッチアップ方法だと思います。Kiro や Amazon Bedrock AgentCoreを実際に触ってみて活用していきたいです。

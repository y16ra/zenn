## Repository Overview

This is a Zenn content repository for managing technical articles and books published on https://zenn.dev/y16ra. Zenn is a Japanese tech blog platform that uses Markdown files stored in Git repositories.

## Content Structure

- **articles/**: Contains individual tech articles as Markdown files (`.md`)
  - Each article is a standalone file named with a unique ID (e.g., `0c58b8e5928ec3.md`)
  - Articles have frontmatter with metadata: title, emoji, type (tech/idea), topics, published status

- **books/**: Contains books composed of multiple chapters
  - Each book is in a subdirectory (e.g., `books/f345863ef0944b/`)
  - `config.yaml`: Book metadata including title, summary, topics, chapters order, price, and published status
  - Individual chapter files (e.g., `introduction.md`, `install-setup.md`)

- **images/**: Static assets for articles and books

## Common Commands

### Using Makefile (Recommended)

The repository includes a Makefile for convenient command execution. Run `make help` to see all available commands:

```bash
# Display help
make help

# Textlint commands
make textlint              # Check all Markdown files
make textlint-articles     # Check only articles
make textlint-books        # Check only books
make textlint-fix          # Auto-fix fixable issues

# Zenn CLI commands
make preview               # Preview content in browser
make new-article           # Create new article
make new-book              # Create new book
make list                  # List all articles and books

# Link checking
make check-links           # Check for broken links
```

### Direct Zenn CLI Commands
```bash
# Preview content locally in browser
npx zenn preview

# Create new article (generates unique ID automatically)
npx zenn new:article

# Create new book (generates unique ID automatically)
npx zenn new:book

# List all articles
npx zenn list:articles

# List all books
npx zenn list:books
```

### Direct Quality Check Commands
```bash
# Run textlint on all articles (日本語文章の校正)
npx textlint "articles/**/*.md"

# Run textlint on a specific article
npx textlint articles/0c58b8e5928ec3.md

# Run textlint on book chapters
npx textlint "books/**/*.md"

# Auto-fix textlint issues where possible
npx textlint --fix "articles/**/*.md" "books/**/*.md"

# Check for broken links (requires npm install first)
npx markdown-link-check articles/*.md
```

## Textlint Configuration

The repository uses textlint for Japanese technical writing quality checks. Configuration is in `.textlintrc`:

- **textlint-rule-no-mix-dearu-desumasu**: Ensures consistent use of です/ます (polite form) or である/だ (plain form) throughout the text
- **textlint-rule-max-ten**: Limits the number of consecutive 、(commas) in a sentence
- **textlint-rule-spellcheck-tech-word**: Validates spelling of technical terms
- **preset-ja-technical-writing**: Japanese technical writing best practices

When editing articles, always run textlint to maintain consistent Japanese writing style.

## CI/CD Workflows

GitHub Actions run automatically on PRs and pushes:

- **textlint.yaml**: Runs textlint on PR articles and posts review comments (triggered on PRs to main)
- **checklinks.yaml**: Validates all Markdown links (runs on push to main and daily at 8:15 AM UTC)
- **pullreq.yaml**: Additional PR checks

## Article Format

When creating or editing articles, use this frontmatter structure:

```yaml
---
title: "記事のタイトル"
emoji: "😊"  # Any emoji
type: "tech"  # "tech" for technical articles, "idea" for idea articles
topics: ["go", "aws"]  # Tags (lowercase, English)
published: true  # false for drafts
---
```

## Book Format

Book `config.yaml` structure:

```yaml
title: "本のタイトル"
summary: "本の説明"
topics: ["ai", "ide"]
published: false  # false for unpublished
price: 0  # 0 for free, or 200-5000 for paid
chapters:
  - chapter-slug-1
  - chapter-slug-2
```

## Development Notes

- All content is in Japanese (日本語)
- Use Zenn CLI for content management operations rather than manually creating files
- Always validate Japanese text with textlint before committing
- Article IDs and book IDs are auto-generated by Zenn CLI - do not create them manually
- Images should be placed in `images/` directory and referenced using relative paths
